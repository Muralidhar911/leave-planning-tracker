# Leave Planning Tracker – Team Web Application

## 1. Project Overview

The Leave Planning Tracker is a web application designed for internal team use to manage and visualize planned leaves. The system allows team members to plan their leaves in advance while enabling the admin to manage users and ensure proper team coverage.

The application will provide:
- Leave planning and tracking
- A shared team leave calendar
- User management by admin
- A modern, visually appealing UI with vibrant colors and smooth animations

The UI should feel modern, clean, and engaging so that team members enjoy using it regularly.

---

# 2. Technology Stack

## Frontend
Next.js (React Framework)
Tailwind CSS
Framer Motion (for cinematic animations)
FullCalendar (team calendar view)
ShadCN UI or Material UI components

## Backend / Database
Supabase (PostgreSQL + Authentication)

## Deployment
Frontend: Vercel
Backend: Supabase Cloud

---

# 3. User Roles

## Admin

Admin has full access to the system.

Admin capabilities:

- Add team members{Name, Email}
- Remove team members
- Assign default passwords
- View all planned leaves
- Access admin dashboard
•	Add Member form: full name, email address, role, default password. Supabase Auth creates the user; the profile row is inserted automatically via a database trigger.

---

## User

Team members who plan their leaves.

User capabilities:

- Login to the application
- Change password
- Add leave plans
- Edit leave plans
- Delete leave plans
- View team leave calendar
•	Provide a shared monthly calendar showing all planned leaves at a glance.

# 5. Database Design

Database: PostgreSQL (Supabase)

---

## Table: users

Columns:

id (uuid)
name (text)
email (text)
role (text)
created_at (timestamp)

---

## Table: leaves

Columns:

id (uuid)
user_id (uuid)
start_date (date)
end_date (date)
reason (text)
created_at (timestamp)

---

Relationship:

users.id → leaves.user_id
